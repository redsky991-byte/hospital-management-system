    Step-by-Step Setup Guide
    1
    Install Node.js

    Download Node.js v18 or later from nodejs.org. Verify with node -v and npm -v.

    2
    Clone the Repository

    git clone <repo-url> then cd hospital-management-system.

    3
    Install Dependencies

    Run npm install in the project directory to install all packages.

    4
    Configure Environment Variables

    Copy .env.example to .env. Set PORT (default 3000) and a strong JWT_SECRET.

    5
    Start the Server

    Run npm start. The console will print: MedCare HMS running on http://localhost:3000.

    6
    First Login

    Open http://localhost:3000. Log in with admin@hospital.com / Admin@123.

    7
    Change the Default Password

    Go to Users, find the admin account, click Edit, and set a new secure password immediately.

    8
    Configure Sites, Wards & Departments

    Go to Settings to add your hospital sites, wards, and clinical departments before registering patients.

    Default Credentials
    Emailadmin@hospital.com
    PasswordAdmin@123
    RoleAdmin
    Change these credentials immediately after first login.

    Patient Management

    Create and manage complete patient records.

    • Add Patient
      Click Add Patient or press N to open the registration form. Fill in demographics, contact info, and blood group.
    • Search Patients
      Use the search box (press /) to filter by name, phone, or ID in real time.
    • Print Patient Record
      Open a patient record and click Print, or press Ctrl+P.
    Appointment Scheduling

    Book and manage patient appointments.

    • New Appointment
      Press N or click New Appointment. Select patient, doctor, department, date, and time slot.
    • View & Filter
      Switch between list and calendar views. Filter by date, doctor, or status.
    • Print Appointment Slip
      Click Print Slip on any appointment row to produce a printable ticket.
    Billing & Invoicing

    Generate invoices and record payments (Admin only).

    • Create Invoice
      Press N to open the invoice form. Add line items, apply discount and tax rates.
    • Record Payment
      Click Record Payment on a pending invoice to update its status to Paid.
    • Print Invoice
      Click Print on any invoice or press Ctrl+P.
    User Management

    Create and manage staff accounts (Admin only).

    • Add User
      Press N. Assign a role (Admin, Doctor, or Nurse) and select a site.
    • Activate / Deactivate
      Toggle a user's status to temporarily prevent login without deleting the account.
    Audit Logs

    Compliance-ready activity trail (Admin only).

    • View Activity
      Every create, update, and delete action is logged with the user's name, timestamp, and affected record.
    • Filter Logs
      Use the search box to filter by action type, user, or resource.
    Settings

    Configure the system structure (Admin only).

    • Sites
      Add hospital branches or facilities. Each site has its own wards and departments.
    • Wards
      Add wards within a site (e.g., General Ward, ICU, Maternity).
    • Departments
      Add clinical departments per site (e.g., Cardiology, Orthopaedics).
    Environment Variables (.env)

    Create a .env file in the project root. Copy from .env.example if available.

    Variable Default Description
    PORT 3000 TCP port the server listens on
    JWT_SECRET required Secret key for signing JWT tokens. Use a random string ≥ 32 characters.
    Generate a strong secret: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

    Production Deployment
    1. Set a strong JWT_SECRET in .env.
    2. Change the default admin password.
    3. Use PM2 as a process manager: npm install -g pm2
      pm2 start server.js --name medcare-hms
      pm2 save && pm2 startup
    4. Place behind Nginx or Apache with HTTPS.
    5. Back up data/hospital.db regularly.

    MedCare HMS is a Node.js application. It requires a server that can run Node.js — it cannot be uploaded as plain PHP/HTML files to a traditional shared host. Choose one of the methods below based on your hosting environment.
    Method A — VPS / Cloud Server (Recommended) Most Reliable

    Works on any Linux VPS: DigitalOcean, Linode, AWS EC2, Azure VM, Google Cloud, Hostinger VPS, etc.

    1
    Connect to Your Server via SSH

    Open a terminal and connect: ssh root@YOUR_SERVER_IP. Replace YOUR_SERVER_IP with your server's public IP address.

    2
    Install Node.js (v18 or later)

    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt-get install -y nodejs     Verify: node -v and npm -v.

    3
    Upload the Project Files

    Option 1 – Git (fastest): git clone https://github.com/redsky991-byte/hospital-management-system.git /var/www/medcare
    cd /var/www/medcare     Option 2 – FTP/SFTP: Use FileZilla or WinSCP. Connect with SFTP (port 22), upload the entire project folder to /var/www/medcare.

    4
    Install Dependencies

    cd /var/www/medcare && npm install --omit=dev

    5
    Create the .env File

    cp .env.example .env
    nano .env     Set PORT=3000 and a strong random JWT_SECRET. Save and close (Ctrl+X, Y, Enter).

    6
    Keep the App Running with PM2

    npm install -g pm2
    pm2 start server.js --name medcare-hms
    pm2 save
    pm2 startup     PM2 automatically restarts the app after a server reboot or crash.

    7
    Open the Firewall Port

    sudo ufw allow 3000/tcp
    sudo ufw allow OpenSSH
    sudo ufw enable     Test by visiting http://YOUR_SERVER_IP:3000 in your browser.

    8
    Point Your Domain to the Server

    Log in to your domain registrar (Namecheap, GoDaddy, Cloudflare, etc.) and add an A Record: Type: A
    Name: @ (or subdomain, e.g. hms)
    Value: YOUR_SERVER_IP
    TTL: Auto (or 300)     DNS changes can take up to 24 hours to propagate.

    9
    Set Up Nginx Reverse Proxy (serve on port 80/443)

    Install Nginx: sudo apt install nginx -y
    Create a site config: sudo nano /etc/nginx/sites-available/medcare Paste the following (replace yourdomain.com): server {
      listen 80;
      server_name yourdomain.com www.yourdomain.com;

      location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
      }
    }     Enable and reload: sudo ln -s /etc/nginx/sites-available/medcare /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx     Now visit http://yourdomain.com — no port number needed.

    10
    Enable Free HTTPS (SSL) with Let's Encrypt

    sudo apt install certbot python3-certbot-nginx -y
    sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com     Follow the prompts. Certbot automatically updates your Nginx config for HTTPS and renews the certificate every 90 days.

    Method B — cPanel Shared Hosting with Node.js Support For cPanel Hosts
    Not all shared hosts support Node.js. Check with your host (e.g., Namecheap, Hostinger, A2 Hosting) that Node.js App Manager is available in cPanel.
    1
    Upload Files via File Manager or FTP

    Zip your project folder (excluding node_modules/ and data/), upload the zip to your hosting home directory using cPanel File Manager or an FTP client (FileZilla), then extract it.

    2
    Open Node.js App Manager in cPanel

    In cPanel, search for Setup Node.js App. Click Create Application.

    3
    Configure the Application

    Fill in the fields:
    • Node.js version: 18 or 20
    • Application mode: Production
    • Application root: Path to your uploaded project folder (e.g., medcare)
    • Application URL: Your domain or subdomain
    • Application startup file: server.js

    4
    Set Environment Variables

    In the same Node.js App Manager form, add environment variables: PORT=3000 and JWT_SECRET=<your-strong-secret>. Save.

    5
    Install Dependencies via SSH or App Manager

    If SSH is available: cd ~/medcare && npm install --omit=dev. Some cPanel Node.js managers have a button Run NPM Install — use that if no SSH access.

    6
    Start the Application

    Click Start App in cPanel Node.js Manager. Visit your domain — MedCare HMS should load.

    Method C — Docker (Advanced)
    1
    Create a Dockerfile in the Project Root

    FROM node:20-alpine
    WORKDIR /app
    COPY package*.json ./
    RUN npm ci --omit=dev
    COPY . .
    EXPOSE 3000
    CMD ["node", "server.js"]

    2
    Build and Run the Container

    docker build -t medcare-hms .
    docker run -d -p 3000:3000 \
      -e JWT_SECRET=your-secret \
      -v $(pwd)/data:/app/data \
      --name medcare-hms \
      medcare-hms     Access at http://YOUR_SERVER_IP:3000. Combine with Nginx (Step 9 above) for a domain + HTTPS.

    Pre-Launch Checklist
    • Strong JWT_SECRET set in .env
    • Default admin password changed
    • HTTPS enabled (Let's Encrypt or paid cert)
    • PM2 or process manager running
    • Firewall configured (block direct port 3000 after Nginx)
    • Regular backups of data/hospital.db scheduled
    • Hospital sites, wards, and departments added in Settings
    Common Deployment Issues
    ProblemFix
    Site shows "502 Bad Gateway" Node app is not running. Run pm2 status and restart: pm2 restart medcare-hms
    HTTPS certificate error Make sure DNS is fully propagated before running Certbot. Check with dig yourdomain.com
    Cannot write to database Ensure the data/ folder exists and is writable: chmod 755 data
    App crashes on start Check logs: pm2 logs medcare-hms. Most common cause: missing .env or wrong Node.js version
    Files uploaded but app won't start on cPanel Ensure node_modules was installed on the server, not uploaded from your local machine
